## 2. Key Modules & Implementations

### A. Data Layer: `src/lib/cms.ts`
Loads list data by fetching the Google Sheet published CSV and dynamically overlaying it with local JSON copy.

*   `getAllReferrals()`:
    1.  Fetches CSV from `${GOOGLE_SHEET_CSV_URL}&_t=${Date.now()}`.
    2.  Parses CSV with `Papa.parse`.
    3.  Extracts arrays (`steps`, `pros`, `cons`) using the `parseSheetArray` helper:
        *   Splits by `|` or newlines `\n` to prevent lists from collapsing into a single paragraph.
        *   Falls back to splitting by `,` for legacy rows.
    4.  Funnels rows into local details fallback if local JSON files contain details.
    5.  Filters out offers where status !== `"active"`.

*   `getReferralBySlug(slug)`:
    Fetches the specific offer row and overlays local JSON text copy, returning a unified `Referral` object.

---

### B. Expiration Scan: `scripts/check_expirations.mjs`
Fires daily during the build workflow to manage offer lifecycles.

1.  Calculates `timeDiff = expiryDate - today`.
2.  **Notification (within 36 hours)**: If `0 < timeDiff <= 36 hours`, it sends a POST request to the Google Webhook containing:
    ```json
    {
      "type": "expiry_alert",
      "name": "Zomato",
      "code": "BPEVW8F",
      "expiry": "7/5/2026",
      "slug": "zomato-promo-code",
      "category": "Food",
      "link": "https://zomato.com/invite..."
    }
    ```
3.  **Auto-Disable (expired)**: If `timeDiff <= 0`, it sends an update request to mark status as `"expired"`.

---

### C. Webhook Handler: `scripts/apps_script_backup.js`
Runs in Google Apps Script inside the user's spreadsheet workspace.

*   `doPost(e)`:
    Handles three action types:
    1.  `expiry_alert`:
        *   Triggers `MailApp.sendEmail` to send a styled HTML email template to `itssaify.17@gmail.com`.
        *   Calls `CalendarApp.createAllDayEvent` to place a reminder event on the default Google Calendar.
        *   Filters out duplicate event titles by scanning calendar schedules (`calendar.getEvents`).
    2.  `update_referral`:
        *   Finds the target row matching the slug or code.
        *   Updates the spreadsheet cells (e.g., changes status to `"expired"` or `"not active"`).
    3.  `feedback`:
        *   Appends user reviews securely to a designated `Feedback` sheet tab.

---

### D. Step Text Formatting: `src/app/[slug]/page.tsx`
Parses the step instructions safely at build time to strip raw markdown/HTML.

*   `cleanStepText(text)`:
    *   Replaces `<a href="...">TEXT</a>` with the raw URL.
    *   Removes wrapper tags (`<strong>`, `<b>`, `<code>`).
    *   Strips duplicate asterisks or backticks.
*   `parseStepText(text, referralCode)`:
    *   Tokenizes the cleaned instruction text.
    *   Replaces download links with interactive anchor links (`target="_blank"`).
    *   Replaces the specific referral code string with the `<InlineCopyBadge />` component.

---

### E. Interactive Copy Badge: `src/components/InlineCopyBadge.tsx`
Renders a custom copyable badge inside text blocks.

*   **Properties**: `code`, `brandName`, `slug`.
*   **Behavior**: Clicking the badge copies the code to the user's clipboard and triggers a micro-animation with a "Copied!" checkmark icon and tooltip.
*   **Analytics**: Fires custom Google Analytics tracking events on copy.
